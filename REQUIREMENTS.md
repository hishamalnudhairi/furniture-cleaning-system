# REQUIREMENTS.md — متطلبات النظام

نظام ويب MVP لمغسلة كنب وزوالي وسجاد.

## 1. المستخدمون والأدوار (Roles)
| الدور | الصلاحيات |
|------|-----------|
| **مدير (Admin)** | كل الصلاحيات + الإعدادات + التقارير. |
| **محاسب (Accountant)** | مراجعة الطلبات، تحديد الأسعار، إصدار الفواتير، محاسبة السائقين. |
| **عامل (Worker)** | تسجيل/متابعة الطلبات عبر واجهة بسيطة. |
| **سائق (Driver)** | استلام/تسليم الطلبات وتحديث حالتها. |
| **عميل (Customer)** | صفحة خارجية لطلب الخدمة (بدون تسجيل دخول، بدون فاتورة). |

## 2. المتطلبات الوظيفية (Functional)

### 2.1 صفحة العميل الخارجية
- نموذج طلب خدمة عام (Public) لا يتطلب تسجيل دخول.
- الحقول: الاسم، رقم الجوال، نوع الخدمة (كنب/زوالي/سجاد)، وصف، ملاحظات.
- **تحديد الموقع على الخريطة = متطلب أساسي (إلزامي)** — يُحفظ خط الطول والعرض.
- **رفع الصور = اختياري**.
- عند الإرسال: يُنشأ طلب بحالة `pending_review` **بدون فاتورة**.
- رسالة تأكيد بسيطة للعميل بعد الإرسال.

### 2.2 سير عمل المحاسب
- يرى الطلبات الواردة بحالة `pending_review`.
- يراجع الطلب ويكمل بياناته.
- **يحدد السعر** ويطبّق الضريبة من الإعدادات.
- **يصدر الفاتورة**.
- الفاتورة تحتوي **QR Code** يفتح **موقع منزل العميل على Google Maps** (من إحداثيات الطلب).

### 2.3 واجهة العامل
- بسيطة جدًا، أزرار كبيرة، خطوات قليلة.
- عرض الطلبات المسندة وتحديث حالتها.

### 2.4 السائقون والتوصيلات
- إسناد طلبات للسائقين.
- تتبّع حالة التوصيل (خرج للتوصيل / تم التسليم).
- **محاسبة السائقين** (مبالغ مستلمة، تسويات).

### 2.5 المخزون البسيط
- أصناف بسيطة (مواد تنظيف/مستلزمات) مع كمية.
- خصم/إضافة يدوي بسيط.

### 2.6 التقارير الأساسية
- عدد الطلبات حسب الحالة.
- إجمالي المبيعات/الفواتير لفترة.
- أداء السائقين (طلبات/مبالغ).
- ملخص المخزون.

### 2.7 الفواتير والطباعة
- إصدار فاتورة لكل طلب مكتمل التسعير.
- QR Code للموقع على الخريطة.
- دعم **طباعة حرارية** (Thermal) قابلة للإعداد.

## 3. صفحة الإعدادات الشاملة (Settings)
يجب أن تشمل:
- اسم المشروع.
- بيانات المحل (عنوان، هاتف، بريد).
- رقم النشاط / السجل.
- أوقات العمل.
- الشعار (Logo).
- البوستر (Poster).
- الألوان (Theme colors).
- الضريبة (نسبة/تفعيل).
- إعدادات الفاتورة (ترقيم، حقول، نص ختامي).
- إعدادات الطباعة الحرارية (عرض الورق، حجم الخط...).
- إعدادات اللغة (اللغة الافتراضية، تفعيل اللغات).
- إعدادات السائقين (عمولات/قواعد المحاسبة).
- إعدادات المخزون.

## 4. المتطلبات غير الوظيفية (Non-Functional)
- **اللغة**: عربي (RTL) + إنجليزي (LTR).
- **التجاوب**: موبايل أولًا.
- **سهولة الاستخدام**: مناسبة لعمّال قليلي الخبرة.
- **الأمان**: تحقق من المدخلات، حماية صفحة العميل من السبام (مستقبلًا)، فصل صلاحيات الأدوار.
- **الأداء**: استجابة سريعة على الجوال.
- **القابلية للصيانة**: كود نظيف يتبع معايير Laravel.

## 5. خارج النطاق (Out of Scope للـ MVP)
- بوابات دفع إلكترونية.
- تطبيق جوال أصلي (Native app).
- إشعارات SMS/Push (يمكن لاحقًا).
- تقارير تحليلية متقدمة.
